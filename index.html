PROJECT_01
Network Automation với Ansible & HSRP
AUTOMATION
⬡ Situation

Môi trường doanh nghiệp cần High Availability cho Default Gateway. Cấu hình thủ công gây ra sai sót, mất nhiều thời gian và thiếu nhất quán giữa các thiết bị trong mô hình mạng 3 lớp.

⬡ Task

Thiết kế và triển khai giải pháp HSRP tự động hóa hoàn toàn bằng Ansible trên Cisco L3 Switch, đảm bảo Idempotency và khả năng Failover tức thời.

⬡ Action

Thiết kế topology 3-layer trên EVE-NG → Viết Ansible Playbook với Jinja2 template để tự động push cấu hình HSRP, IP, Interface → Thiết lập Priority/Preempt để kiểm soát Active/Standby → Test Failover bằng cách ngắt thiết bị Active và quan sát convergence time.

Key Implementation
hsrp_playbook.yml
# Ansible Playbook — HSRP Auto-Configuration
---
- name: Configure HSRP on L3 Distribution Switches
  hosts: distribution_switches
  gather_facts: no
  vars_files:
    - vars/network_vars.yml

  tasks:
    - name: Configure SVI Interface
      cisco.ios.ios_config:
        lines:
          - "ip address {{ item.ip }} {{ item.mask }}"
          - "no shutdown"
        parents: "interface {{ item.vlan_int }}"
      loop: "{{ svi_interfaces }}"

    - name: Apply HSRP Configuration (Jinja2 Template)
      cisco.ios.ios_config:
        src: templates/hsrp.j2
      notify: Save Config

  handlers:
    - name: Save Config
      cisco.ios.ios_command:
        commands: write memory
Outcomes
✓ Results

Giảm 90% thời gian cấu hình thủ công — từ ~45 phút xuống còn ~4 phút cho toàn bộ stack. Đảm bảo Idempotency (chạy nhiều lần không gây lỗi cấu hình). Gateway Failover được kiểm chứng với thời gian hội tụ dưới 3 giây. Playbook có thể tái sử dụng cho bất kỳ môi trường tương tự nào chỉ bằng cách thay đổi file biến.

PROJECT_02
Wi-Fi Security — RADIUS 802.1X & Evil-Twin Lab
SECURITY
⬡ Situation

Wi-Fi doanh nghiệp sử dụng PSK (Pre-Shared Key) dễ bị tấn công Evil-Twin và credential theft. Không có cơ chế xác thực tập trung hay audit log truy cập.

⬡ Task

Triển khai hạ tầng xác thực 802.1X tích hợp Active Directory và kiểm chứng hiệu quả phòng thủ bằng cách tự thực hiện tấn công Evil-Twin trong môi trường lab kiểm soát.

⬡ Action

Cấu hình Windows Server NPS làm RADIUS server → Tích hợp AD DS để xác thực user domain → Triển khai 802.1X với EAP-TLS/PEAP và cấp Digital Certificate qua CA nội bộ → Mô phỏng Evil-Twin bằng Kali Linux (Hostapd-mana + Dnsmasq) → Phân tích traffic và xác nhận certificate validation block cuộc tấn công.

PROJECT_03
Web Infrastructure — LEMP Stack Hardening
SYSADMIN
⬡ Situation

Hệ thống web nội bộ cần được triển khai trên Linux với bảo mật cao, tránh các cuộc tấn công brute-force phổ biến và có khả năng tự phục hồi dữ liệu khi sự cố xảy ra.

⬡ Task

Build và harden toàn bộ LEMP Stack từ zero trên Ubuntu Server, áp dụng nguyên tắc Principle of Least Privilege và Whitelist-only access policy.

⬡ Action

Cài đặt Nginx + MariaDB + PHP trên Ubuntu Server → SSH Hardening (đổi port, disable password auth, SSH Key only) → Cấu hình UFW theo Whitelist policy → Deploy Fail2Ban với ruleset tùy chỉnh → Viết Bash script tự động backup daily với Cronjob.

Auto-Backup Script
backup.sh
#!/bin/bash
# Daily Backup Script — DB + Source Code
# Cronjob: 0 2 * * * /opt/scripts/backup.sh

TIMESTAMP=$(date +%Y%m%d_%H%M%S)
BACKUP_DIR="/var/backups/webapp"
WEBROOT="/var/www/html"
DB_NAME="production_db"
RETAIN_DAYS=7

# Create backup directory
mkdir -p "${BACKUP_DIR}/${TIMESTAMP}"

# Dump MariaDB
mysqldump --single-transaction "${DB_NAME}" \
  | gzip > "${BACKUP_DIR}/${TIMESTAMP}/db.sql.gz"

# Archive source code
tar -czf "${BACKUP_DIR}/${TIMESTAMP}/webroot.tar.gz" \
  -C "${WEBROOT}" .

# Remove backups older than RETAIN_DAYS
find "${BACKUP_DIR}" -maxdepth 1 -mtime +"${RETAIN_DAYS}" \
  -exec rm -rf {} \;

echo "[OK] Backup completed: ${TIMESTAMP}"
Outcomes
✓ Results

Hệ thống chịu đựng thành công các cuộc tấn công port scan và brute-force trong môi trường test (Fail2Ban block 100% sau 5 lần thử). Backup tự động chạy hàng ngày lúc 2AM, lưu trữ 7 ngày rolling. Recovery time từ backup dưới 5 phút. SSH attack surface giảm đáng kể nhờ custom port + key-only auth.
